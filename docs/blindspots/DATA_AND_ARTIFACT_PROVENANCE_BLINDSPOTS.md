# Data and Artifact Provenance Blindspots

This document exposes artifact ambiguity, unclear dataset provenance, and contamination pathways.

- Blindspot ID: BS-DAT-001
- Title: Artifact Provenance Ambiguity
- Category: Data and Artifact Provenance
- Severity: medium
- Confidence: low
- Status: weak suspicion
- Why this is a blindspot: The root directory contains multiple `.jsonl` files (e.g., `research_data.jsonl`, `research_data2.jsonl`, `dpo_correction_dataset_v2.jsonl`, `dpo_correction_dataset_v4.jsonl`). It is entirely unclear whether these are historical backups, actively consumed inputs, or outputs from the current `analyze_results.py` pipeline.
- Evidence:
  - `docs/LEGACY_ARTIFACT_INVENTORY.md` lists these artifacts.
  - Scripts like `create_preference_dataset.py` might silently depend on these exact filenames.
- What could be misleading: A developer or automated agent might assume `research_data.jsonl` is the canonical output of a recent sprint and base their optimizations on it, when in reality it might be corrupted or generated by a divergent version of the agent (e.g., the notebook version).
- Potential consequence: Optimization cycles and DPO training might be based on noisy, outdated, or fundamentally flawed artifact data, polluting the results.
- How to verify or falsify it: Trace the file `dpo_correction_dataset.jsonl` through all active pipeline scripts to determine if it is generated or consumed.
- Recommended next step: Move generated datasets to a dedicated `data/processed/` or `data/raw/` directory and update scripts to use parameterized paths rather than hardcoded root paths.
- Blocks future work? no

- Blindspot ID: BS-DAT-002
- Title: Unstructured Text Dumps in Root
- Category: Data and Artifact Provenance
- Severity: low
- Confidence: high
- Status: fact
- Why this is a blindspot: The root directory contains numerous text files and transcripts (`BrowserChat.txt`, `GeminiChat.txt`, `pdf1.txt`, `pdf_abstracts.txt`) that appear to be manual inputs or debug output.
- Evidence:
  - `docs/LEGACY_ARTIFACT_INVENTORY.md` categorizes these files.
- What could be misleading: These files might be mistaken for important configuration or data dependencies, creating mental overhead.
- Potential consequence: They obscure the active data pipeline and create confusion around the project's actual source of truth for problem statements.
- How to verify or falsify it: Use `grep` to ensure these filenames are not hardcoded into any Python scripts.
- Recommended next step: Follow the recommendations in `docs/MOVE_CANDIDATES.md` to cleanly separate research materials and legacy noise into an `_attic/noise/` directory.
- Blocks future work? no
